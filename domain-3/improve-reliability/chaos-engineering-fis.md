# Chaos Engineering (FIS)

## Overview
AWS Fault Injection Simulator (FIS) is a managed service for performing chaos engineering experiments.

## Key Features
- **Fault Injection**: Controlled failures
- **Experiment Templates**: Reusable experiments
- **Safety Controls**: Stop conditions and rollbacks
- **Integration**: Service integration

## Use Cases
- Resilience testing
- Failure simulation
- Recovery testing
- Architecture validation

## Benefits
- Improved resilience
- Failure preparedness
- Better understanding
- Risk reduction

## Experiment Types
- **API Faults**: Throttling, errors
- **Infrastructure Faults**: Instance termination
- **Network Faults**: Latency, packet loss
- **Custom Faults**: Lambda functions

## Integration
- EC2 instances
- ECS/EKS containers
- Lambda functions
- RDS databases
