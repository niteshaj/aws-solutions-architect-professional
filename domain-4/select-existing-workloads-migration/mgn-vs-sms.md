# MGN vs SMS

## Overview
AWS Application Migration Service (MGN) and Server Migration Service (SMS) provide different approaches for server migration.

## Application Migration Service (MGN)
- **Continuous Replication**: Real-time data sync
- **Minimal Downtime**: Near-zero downtime migration
- **Automated Testing**: Test migrations before cutover
- **Broad Support**: Windows and Linux servers

## Server Migration Service (SMS)
- **Image-based**: VM image replication
- **Scheduled Migrations**: Planned migration windows
- **Incremental Updates**: Efficient replication
- **Simple Setup**: Easy configuration

## Use Cases
- **MGN**: Continuous replication, minimal downtime
- **SMS**: Planned migrations, image-based approach

## Key Differences
- **Replication**: Continuous vs scheduled
- **Downtime**: Minimal vs planned
- **Complexity**: Advanced vs simple
- **Testing**: Advanced vs basic

## Selection Criteria
- Downtime requirements
- Application complexity
- Migration timeline
- Team expertise

## Integration
- Migration Hub
- Application Discovery
- Cost Explorer
- Monitoring tools
