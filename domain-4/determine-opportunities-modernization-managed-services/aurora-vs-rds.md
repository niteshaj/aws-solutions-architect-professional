# Aurora vs RDS

## Overview
Amazon Aurora is a cloud-native relational database, while RDS provides managed traditional databases.

## Aurora Benefits
- **Performance**: 5x faster than MySQL, 3x faster than PostgreSQL
- **Scalability**: Auto-scaling storage
- **High Availability**: Multi-AZ with instant failover
- **Serverless**: Aurora Serverless option

## RDS Benefits
- **Compatibility**: MySQL, PostgreSQL, SQL Server, Oracle, MariaDB
- **Familiarity**: Traditional database engines
- **Migration**: Easier migration from on-premises
- **Cost**: Potentially lower for simple workloads

## Use Cases
- **Aurora**: High performance, scalability, cloud-native
- **RDS**: Compatibility, familiarity, simple workloads

## Key Differences
- **Storage**: Aurora auto-scaling vs RDS provisioned
- **Performance**: Aurora optimized vs RDS standard
- **Features**: Aurora advanced features vs RDS basic
- **Cost**: Aurora higher performance vs RDS lower cost

## Selection Criteria
- Performance requirements
- Compatibility needs
- Scalability requirements
- Budget constraints

## Integration
- Migration tools
- Monitoring services
- Backup solutions
- High availability
