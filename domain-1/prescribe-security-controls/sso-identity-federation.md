# SSO & Identity Federation

## Overview
AWS Single Sign-On (SSO) and Identity Federation enable centralized user management and access control across AWS accounts and applications.

## AWS SSO Features
- Centralized user directory
- Multi-account access
- Application integration
- Permission sets
- User and group management

## Identity Federation Types

### SAML 2.0 Federation
- Enterprise identity providers
- Active Directory integration
- SSO for web applications
- Attribute-based access control

### OpenID Connect (OIDC)
- Web and mobile applications
- API access
- Social identity providers
- Custom authentication

### Federation with External IdPs
- Azure AD
- Okta
- Ping Identity
- Custom SAML providers

## Components

### AWS SSO Directory
- Built-in user store
- External directory integration
- Group management
- Attribute synchronization

### Permission Sets
- Role-based access control
- Multi-account permissions
- Session duration controls
- Access policies

### Application Assignments
- AWS account access
- Cloud applications
- Custom applications
- User and group assignments
